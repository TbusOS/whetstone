What a distilled skill looks like.蒸出来的 skill 长什么样。

A whetstone skill is a portable, runtime-neutral package: a SKILL.md the runtime selects on, platform-specific values quarantined in params/<platform>.md, and pitfalls split out. Here's the anatomy, the kinds of skills people distill, and how to use one. All examples below are generic — real skills carry your own platform values. whetstone 的 skill 是一个可移植、runtime 中立的包：一份 runtime 据以选择的 SKILL.md、隔离到 params/<platform>.md 的平台具体值、单独拆出的坑库。下面讲它的解剖、能蒸哪类 skill、怎么用。例子全是通用占位 —— 真实 skill 带你自己的平台值。

Usage用法

How to use a skill.怎么用一个 skill。

  1. Trigger触发: say the skill's trigger phrase; the runtime matches on the description and loads the body.说出该 skill 的触发词，runtime 按 description 匹配并加载正文。
  2. Swap platform换平台: new chip / SDK version → add one params/<platform>.md; the L1/L2 body stays put.换芯片 / SDK 版本 → 加一份 params/<platform>.md，L1/L2 正文不动。
  3. Keep the index clean索引卫生: after adding / editing a skill run whetstone lint (overlaps / collisions) and whetstone index (grouped catalog).新增 / 改 skill 后跑 whetstone lint（重叠 / 撞车）和 whetstone index（分族目录）。
  4. Move machines跨机部署: keep your real skills in a private repo; clone + deploy into the runtime's skills folder.真实 skill 放私有仓；clone + deploy 进 runtime 的 skills 目录。

Real skills carry internal values, so keep them private — never on public Pages. This page is the generic, shareable view.真实 skill 带内部值，放私有仓、绝不上公开 Pages。本页是通用的可公开版。
