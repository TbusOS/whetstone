Why Cangjie为什么需要它

Your hard-won experience keeps evaporating.你辛苦攒的经验，每次都在蒸发。

You finish a feature. The design, the method, the pitfalls — all of it lives in one throwaway session. Next platform, you re-explain it. A teammate? They can't get it at all. Cangjie fixes the three leaks. 你做完一个功能。那套设计、方法、踩过的坑 —— 全在一次用完即弃的会话里。换个平台，你又从头讲一遍。同事呢？根本拿不到。Cangjie 堵住这三个漏洞。

Gets smarter越用越聪明

Improve the method once, every platform benefits.方法改一次，所有平台一起受益。

The transferable design and pitfalls (L1/L2) live in the skill body. Learn a better diagnosis on platform B and the skill improves for A and C too. The library compounds instead of resetting. 可迁移的设计和坑（L1/L2）留在 skill 正文。在平台 B 学到更好的诊断，平台 A、C 的能力一起变强。库在累积，而不是每次归零。

No re-explaining换平台不重讲

New chip? Swap one params table.换芯片？只换一张参数表。

Platform-specific values (addresses, key lengths, tool names) live in params/<platform>.md — never baked into the prose. A new platform is a new table; the design and method stay untouched. 平台专属的值（地址、密钥长度、工具名）放在 params/<platform>.md —— 从不写死进正文。换平台就新增一张表，设计和方法一个字不动。

Hand it over打包即传能力

Give the package; they get the capability.给出包，对方就有了能力。

The output is a self-contained, runtime-neutral folder. Zip it, hand it to a colleague on Cursor or Codex, they drop it in — and they now have your secure-boot bring-up know-how. No onboarding call. 产物是一个自包含、runtime 中立的目录。打包，递给用 Cursor 或 Codex 的同事，拖进去 —— 他立刻拥有你那套安全启动的经验。不用开会传授。

Architecture架构

A distiller, a portable package, optional sinks.一个蒸馏器，一个可移植包，几个可选 sink。

Runtime-specific code is isolated to a thin capture adapter. Everything you share — the skill package — is plain, neutral markdown. engram and llm-wiki are optional downstream sinks, never required. runtime 专属代码只隔离在薄薄一层采集适配器里。你对外分享的东西 —— skill 包 —— 是纯净中立的 markdown。engram、llm-wiki 是可选下游，从不强制。

The ruler那把尺子

One ruler: how far does this carry?一把尺子：这条知识能走多远？

The wider a piece of knowledge applies, the higher its layer — and the closer it lives to the reusable skill body. The narrower, the more it belongs in a swappable table. 一条知识适用范围越宽，层级越高，越靠近可复用的 skill 正文；越窄，越该放进可替换的参数表。

How it works怎么工作

Mine. Layer. Reconcile. You approve.挖掘。分层。对账。你点头。

Two human gates bracket an automatic middle. Nothing reaches the live library without your sign-off — because an LLM's self-judgment of skill quality is only ~46% accurate. 两道人审关卡夹着一段自动流程。没有你点头，东西进不了正式库 —— 因为 LLM 自评 skill 质量准确率只有约 46%。

A real scenario一个真实场景

Secure boot, done once — owned by everyone.安全启动做一次，人人都有。

  1. You finish secure-boot bring-up on one SoC — keys, OTP, the signing chain, and a nasty buffer-overrun pitfall you only found by dumping bytes.你在一颗 SoC 上做完安全启动 —— 密钥、OTP、签名链，还有一个靠 dump 字节才揪出来的缓冲区越界坑。
  2. Run cangjie distill. It separates the transferable design + pitfall (L1/L2) from this chip's addresses and key length (L3). You approve at two gates.跑 cangjie distill。它把可迁移的设计+坑（L1/L2）和这颗芯片的地址、密钥长度（L3）分开。你在两道关卡点头。
  3. Out comes a verified-boot/ package — neutral markdown, self-contained.产出一个 verified-boot/ 包 —— 中立 markdown，自包含。
  4. A teammate on Cursor (or Codex) drops it into their skills folder. They now have your secure-boot know-how — design, method, and the byte-dump pitfall — instantly.用 Cursor（或 Codex）的同事把它拖进 skills 目录。他立刻拥有你那套安全启动经验 —— 设计、方法、那个 dump 字节的坑。
  5. Next SoC? Add one params/soc-b.md. The design and pitfalls carry over untouched. Optionally sync the whole thing into engram, or publish a wiki page via llm-wiki.下一颗 SoC？加一份 params/soc-b.md。设计和坑原样复用。可选：整包同步进 engram，或用 llm-wiki 发一篇 wiki 页。
Optional companions可选搭档

Plays well with others — needs none of them.能配合，但谁都不强制。

Optional可选

engram

Sync packages into a local memory store that handles dedup, decay and recall. If absent, your runtime's native skill triggering still works.把包同步进本地记忆库，交给它做去重、衰减、召回。没装，靠 runtime 原生触发也能用。

Optional可选

llm-wiki

Publish the distilled knowledge as a human-browsable wiki page — for reading, not loading. A distinct layer, not a duplicate.把蒸馏出的知识发成人看的 wiki 页 —— 给人读，不是给 agent 装。是另一层，不是重复。

Optional · 3rd-party可选 · 第三方

darwin-skill

Score and hill-climb an individual skill's quality over time. Cangjie grows the library; darwin polishes single skills.给单个 skill 打分、逐步进化。Cangjie 让库长大，darwin 打磨单个 skill。
